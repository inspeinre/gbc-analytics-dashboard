Проект интеграции RetailCRM с аналитическим дашбордом на базе Supabase и системой Telegram-уведомлений. Разработано с активным использованием AI-ассистентов (LLM).

## 💡 О проекте

Основная цель проекта — автоматизировать выгрузку заказов из RetailCRM в базу данных Supabase для последующей аналитики, а также настроить умные уведомления о крупных заказах. Так как в используемой версии RetailCRM отсутствовал встроенный функционал Webhooks, была разработана кастомная система синхронизации через систему Триггеров и Serverless-функции на Vercel.

## 🚀 Реализованный функционал

*   **Импорт данных:** Загрузка начальной базы заказов из JSON в RetailCRM через Python API.
*   **Синхронизация CRM $\leftrightarrow$ Supabase:** 
    *   Реалтайм-обновление при создании или изменении заказа.
    *   Ручная синхронизация пакетами по 100 записей через эндпоинт `?manual=true`.
*   **Telegram-уведомления:** Автоматическая отправка сообщений в бота при сумме заказа от `50 000 ₸`.
*   **Кастомные поля в CRM:** 
    *   `orderType`: `eshop-individual` (Интернет-заказ физлица).
    *   `custom_field`: `utm_source` (отслеживание источников трафика).

### Интерактивный Дашборд (React)
*   **Трехпозиционная сортировка таблиц:** Логика реализована в классическом React-стиле на основе состояний (state-driven). Клики по заголовкам работают циклично: *По возрастанию $\rightarrow$ По убыванию $\rightarrow$ Сброс сортировки*.
*   **Умная фильтрация (Toggle-фильтры):** Графики выступают в роли интерактивных кнопок. Повторный клик по активному сегменту графика (например, городу или UTM-метке) автоматически сбрасывает фильтр.
*   **Динамический сброс состояний:** Кнопка «Сбросить фильтры» рендерится в DOM только в том случае, если начальное состояние изменено (выбран любой фильтр или применена сортировка), позволяя очистить всю выборку в один клик.

## 🏗 Архитектура и решения

### 1. Триггеры вместо Webhooks
Из-за ограничений версии CRM синхронизация реализована через механизм автоматизации (Триггеры). При любом изменении заказа CRM отправляет HTTP-запрос на Vercel API, который форматирует данные и делает `upsert` в Supabase.

### 2. Vercel Proxy для Telegram
Из-за региональных сетевых ограничений (отсутствие прямого доступа к API Telegram из RetailCRM) был написан дополнительный API-мост на Vercel. CRM отправляет параметры триггера на Vercel, а тот переадресует их в Telegram-бота.

### 3. API Эндпоинты
*   `/api/sync` — Принимает вебхуки от триггеров CRM и пишет в Supabase.
*   `/api/sync?manual=true` — [Запуск ручной пакетной синхронизации](https://gbc-analytics-dashboard-1.vercel.app/api/sync?manual=true).

## 🤖 Опыт использования AI и LLM

Проект написан с минимальным опытом работы со стеком (кроме Supabase) благодаря использованию ИИ. Фронтенд (HTML/CSS) и логика JS сгенерированы практически полностью через промптинг.

**Используемые модели и инструменты:**
*   **Antigravity / Gemini 3.1 Pro:** Основная модель для написания и рефакторинга кода.
*   **Gemini 3 Flash:** Использовалась для брейншторма, быстрых уточнений и обсуждения идей.
*   **gemma-4-31b-it:** Тестировалась, но показала неудовлетворительные результаты для данной задачи.
*   **Google AI Studio Gemini 3.1 Pro (Preview):** Использовалась как основная веб-среда (все аккаунты на Free-тарифах).

**Сложности при работе с ИИ:**
> *Главной проблемой при использовании бесплатных AI-агентов стало отсутствие нормальной памяти контекста и "галлюцинации". ИИ периодически ломал уже правильно работающие скрипты, внося ненужные усложнения, и допускал синтаксические ошибки при переписывании кода.*
